# 🔥 HLC 챌린지 자동화 — 설계 문서

작성일: 2026-07-06
상태: 승인됨 (구현 계획 단계로 진행)

## 배경 / 목표

친구 3인(이준호·종서·명근)의 자기계발 습관 챌린지 "HLC season 1"(기간 6/22~7/31)은
현재 Notion에서 **전부 수기 관리**된다: 계획 카드 업로드, 완료 상태 변경, 실패 판정, 벌금 집계가
모두 사람 손을 거친다. 이로 인해 (1) 참여자는 상태 변경이 귀찮고 (2) 관리자는 누가 지켰는지
추적이 안 된다.

**목표**: 부재(미제출) 자동 실패 처리 · 완료 자동 판정 · 벌금 자동 집계 · 매일 리포트 알림.

## 선택한 접근 — "Notion에 두뇌만 붙인다" (하이브리드)

```
[Notion]  데이터·입력·인증사진·UI (그대로 유지)
   ▲ │  Notion API (read/write)
   │ ▼
[GitHub Actions cron]  파이썬 스크립트 = 판정/집계 두뇌
   │
   ▼
[Discord Webhook]  8시 판정 리포트
```

- Notion 완전 순정은 "부재 감지(8시까지 안 올린 사람)"가 불가 → 두뇌 필요.
- 새 앱/봇 이주는 3인 규모에 오버스펙 + 기존 데이터/UI 폐기 → 기각.
- GitHub Actions: 무료, 서버 관리 0, 형님 PC 꺼져 있어도 정시 실행.

## 하루 타임라인 & 규칙

challenge day = **D**.

| 시점(KST) | 자동 동작 |
|-----------|-----------|
| D-1 저녁 ~ D 08:00 | 멤버가 D 계획 카드 업로드 (계획 완료) |
| D 낮 ~ 밤 | 멤버가 할 일 체크박스 채움 → 자동 '인증 완료' 반영 (30분 주기 sync) |
| **D 08:00** | ⚖️ 심판 실행 + Discord 리포트 발송 |

### D 08:00 심판 실행 (하루 1회, 핵심)
1. **D-1 마감**: D-1 카드의 "할 일 체크리스트"가 전부 체크 안 됨 → `실패` (인증 미이행, +3,000)
2. **D 계획 점검**: D 카드를 안 올린 멤버 → `실패` (계획 미제출, +3,000)
3. 멤버별 실패 개수 × 3,000원 재집계 → Discord 리포트 발송

## 핵심 메커니즘

### 1) challenge day 자동 판정 (기존 결함 해결)
현재 `날짜` 필드는 전부 비어 있음(제목에 수기). 자동화를 위해:
> 카드 `createdTime`(UTC) → KST 변환 → **다음 08:00 KST로 올림** = 그 카드의 challenge day.
> 스크립트가 `날짜` 필드에 자동 기입.

규칙 "8시 이후 업로드는 다음날로 집계"와 정확히 일치. 멤버는 날짜 수기 불필요.

### 2) 완료 자동 판정
"할 일 체크리스트" 헤딩 아래 to-do 블록이 **전부 checked** → 상태 `인증 완료`로 자동 변경.
사진은 선택. 낮 시간대 30분 주기 sync 잡으로 실시간 느낌 제공.

### 3) 벌금 집계
별도 저장 없이 매 실행마다 보드에서 멤버별 `실패` 수 × 3,000원 즉석 계산 → 리포트.

## Discord 리포트 예시
```
📅 7/6 HLC 리포트
어제(7/5)  ✅ 이준호  ✅ 명근  ❌ 종서 (인증 미이행 +3,000)
오늘(7/6)  ✅ 이준호  ✅ 종서  ❌ 명근 (계획 미제출 +3,000)
💰 누적  준호 6,000 · 종서 3,000 · 명근 3,000  (회식비 창고 12,000원)
```

## 리스크 (확정된 처리)
- 🔴 체크박스 2종 혼동 → **템플릿 단순화**: "인증 체크" 섹션 체크박스 제거, 안내문으로. 체크박스는 할 일에만.
- 🟡 "계획 수정 불가" 강제 불가 → v1 명예제. 8시 이후 수정 시 `⚠️ 수정 흔적`만 리포트에 표시(벌금 X).
- 🟡 타임존 → 크론 23:00 UTC(=08:00 KST). `zoneinfo`로 변환.
- 🟢 멤버 이름 매핑 → user ID 2개 확보(`88de5147`, `baf68de0`), 이름만 config 고정.

## 기각한 접근
- **Notion 완전 순정**: 부재 자동 실패·관리 대시보드 불가 → 핵심 요구 미충족.
- **카톡/디스코드 봇으로 완전 이주**: 데이터·인증사진 UI 폐기, 3인 규모에 과함.

## 구현 전 준비물 (prerequisites)
1. Notion **내부 통합(Internal Integration) 토큰** — GH Actions 스크립트 전용 (MCP OAuth와 별개).
   해당 통합을 "HLC season 1" 페이지에 연결(공유)해야 함.
2. **Discord webhook URL** (알림 채널).
3. **GitHub 저장소** (public=Actions 무제한, private=월 2000분 내). 토큰/웹훅은 Secrets에 보관.
4. 종서/명근 ↔ user ID 매핑 확정.
